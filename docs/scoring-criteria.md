# MDS AI判定 採点基準書

## 概要

本システムでは、社員が提出したミッション内容をAI（Gemini）が6つの観点から1〜10点で採点し、加重計算によって総合ポイント・グレード・グレード給を算出します。

---

## 採点の6項目

### 1. difficulty（難易度）｜加重: 20%

| 点数 | 基準 |
|------|------|
| 1-2 | 定型的・ルーティン業務の範囲 |
| 3-4 | 一部判断が必要だが前例がある |
| **5-6** | **工夫や調整が必要（標準）** |
| 7-8 | 前例が少なく、自ら設計・構築が必要。AI・DX技術の選定・導入設計・組織への定着推進を含む場合はこの水準に該当しやすい |
| 9-10 | 業界でも前例が少なく、高難度な推進が必要（極めて稀） |

### 2. scope（影響範囲）｜加重: 15%

| 点数 | 基準 |
|------|------|
| 1-2 | 自分の業務のみ |
| 3-4 | チーム内に留まる |
| **5-6** | **部署単位に影響（標準）** |
| 7-8 | 複数部署に明確に影響 |
| 9-10 | 全社・事業レベルの影響（極めて稀） |

### 3. innovation（新規性）｜加重: 20%

| 点数 | 基準 |
|------|------|
| 1-2 | 既存運用の延長・前年踏襲 |
| 3-4 | 既存業務の小改善 |
| **5-6** | **新しい工夫を含む（標準）** |
| 7-8 | 新しい仕組み・プロセスの構築。AI・DXツールの活用や自動化の仕組み導入を含む場合は積極的に該当と判断 |
| 9-10 | 組織や競争力に大きく影響する変革（極めて稀） |

### 4. contribution（課題寄与）｜加重: 20%

※ この項目では「対処療法か根治療法か」の観点も評価します。発生した事象への一時的対応に留まるか、原因を特定して再発防止・抜本的解決に取り組んでいるかを見ます。

| 点数 | 基準 |
|------|------|
| 1-2 | 課題との関連が薄い・寄与が小さい。発生事象への一時的対応に留まる |
| 3-4 | 一定の寄与があるが限定的。対処療法的で根本原因への踏み込みがない |
| **5-6** | **主要課題に一定寄与（標準）。表層だけでなく一定の原因分析がある** |
| 7-8 | 重要な経営課題に強く寄与。原因を特定し再発防止・抜本的解決に取り組んでいる。業務効率化・生産性向上・コスト削減をAI・DX・自動化で実現する取り組みはこの水準に該当しやすい |
| 9-10 | 会社の最重要課題の根本原因に切り込み、仕組みレベルで解決に直結（極めて稀） |

### 5. roleLevel（立ち位置・主体的関与）｜加重: 15%

※ この項目では「自ら手を動かしているか」を重視します。肩書きや役職が高くても、部下に仕事を丸投げして自身の具体的な行動が見えない場合は低評価（1-2点）になります。管理職ほど「自分が何をするか」を具体的に示す必要があります。

| 点数 | 基準 |
|------|------|
| 1-2 | 補助的・サポート役。または指示・管理のみで自らの実質的関与が見えない（丸投げ状態） |
| 3-4 | 実行担当として動くが主導権はない。管理職の場合、部下に任せきりで自分の行動が不明確 |
| **5-6** | **主担当として責任を持ち、自ら手を動かしている（標準）** |
| 7-8 | 推進責任者として、方針策定・意思決定・障害排除など自ら主体的に動いている |
| 9-10 | 最終意思決定責任者として、自ら先頭に立ち困難な判断や交渉を行っている（極めて稀） |

### 6. feasibility（実現可能性・計画の具体性）｜加重: 10%

※ この項目ではゴールだけでなく「具体的な計画・手順があるか」を重視します。やりたいことだけ書いて、いつ・何を・どう進めるかが不明確な場合は低評価になります。

| 点数 | 基準 |
|------|------|
| 1-2 | 根拠がほとんど示されていない。ゴールだけの記載で具体的な計画や手順がない |
| 3-4 | 根拠がやや弱い・抽象的。やりたいことは書かれているが、いつ・何を・どう進めるかが曖昧 |
| **5-6** | **一定の根拠あり（標準）。実行計画の骨格はあるが詳細はやや不足** |
| 7-8 | 実績・能力・資源面で具体的根拠がある。具体的なステップ・スケジュール・数値目標が示されている。AI・DXツールや自動化技術を実現手段として具体的に示している場合、計画の実現根拠が強いと判断 |
| 9-10 | 高い蓋然性で実現可能。緻密な計画と明確な根拠があり、リスク対策まで言及（極めて稀） |

---

## 採点方針

- **5点が標準**。大半のミッションは4〜6点に収まる想定です。
- **7点以上**は「明確に優れている」場合のみ。具体的な根拠の記述が必要です。
- **9〜10点**は「極めて例外的」な場合のみ。全体の5%未満が目安です。
- 記述が曖昧・抽象的・具体性に欠ける場合は **1〜3点** に減点されます。
- 「〜を推進する」「〜に貢献する」のような漠然とした表現だけでは5点以上になりません。
- **丸投げ検出**: 上位役職者のミッションで「部下に指示する」「管理する」のみで自身の具体的行動が見えない場合、roleLevel は1〜2点になります。肩書きではなく実際の関与度を評価します。
- **「指導・教育」名目の丸投げ**: 「部下の指導・教育を行う」「部下を育成する」という記載が、実質的に自分のミッションを部下にやらせているだけの場合があります。本当の指導・教育であれば育成の具体的な計画・方法・目標が記載されているはずです。それがない場合は roleLevel 1〜2点になります。

## 戦略的加点方針（AI・DX推進）

当社はAI活用・DX推進・業務効率化・生産性向上を経営戦略上の最重要テーマと位置づけています。ミッション内容にAI・DX・自動化・RPA・データ活用・業務効率化・生産性向上の要素が含まれる場合、以下の項目で積極的に加点します：

| 対象項目 | 加点の考え方 |
|---------|------------|
| innovation（新規性） | AI・DXツールの導入や活用は、新しい仕組み・プロセスの構築に該当し、7点以上の根拠となる |
| contribution（課題寄与） | 効率化・生産性向上による経営課題への寄与は、根本的な解決アプローチとして高く評価 |
| difficulty（難易度） | AI・DX技術の選定・導入・定着には専門的な知識と設計力が必要であり、適切に加点 |
| feasibility（実現可能性） | AI・DXツールや自動化技術を具体的な実現手段として示している場合、計画の実現根拠が強化されていると判断し加点 |

**注意**: 単に「AIを使う」「DXを推進する」と書いてあるだけで具体性がない場合は加点しません。具体的なツール名、適用対象、期待効果が記述されている場合に加点対象とします。

## 加重配分

| 項目 | 加重 |
|------|------|
| difficulty（難易度） | 20% |
| scope（影響範囲） | 15% |
| innovation（新規性） | 20% |
| contribution（課題寄与） | 20% |
| roleLevel（立ち位置） | 15% |
| feasibility（実現可能性） | 10% |
| **合計** | **100%** |

---

## 算出フロー

### Step 1: ミッション基礎ポイント

```
基礎ポイント = difficulty×0.20 + scope×0.15 + innovation×0.20
             + contribution×0.20 + roleLevel×0.15 + feasibility×0.10
```

最大値: 10.0（全項目10点の場合）

### Step 2: ミッション加重ポイント

```
加重ポイント = 基礎ポイント × (ウェイト / 100)
```

例: 基礎ポイント 5.0、ウェイト 30% → 加重ポイント = 1.50

### Step 3: 総合ポイント

```
総合ポイント = 全ミッションの加重ポイントの合計
```

### Step 4: グレード判定

総合ポイントを役職グループ別の閾値テーブルで照合し、グレード（1〜10）を決定。

### Step 5: グレード給

グレードに対応する給与額を役職グループ別テーブルから取得。

| M4 | 関係先、巻き込む相手、自分の役割や立ち位置 |
| M5 | ミッション完遂の根拠 |
